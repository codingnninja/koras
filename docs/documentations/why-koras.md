# Koras: Hybrid UI Component & Rendering System

**Koras** is a no-build, JSX-like UI system that runs directly in browsers, servers, or workers without a virtual DOM or tagged templates.

It lets you render JavaScript components with minimal overhead while keeping the mental model simple and flexible.

---

## Why Koras?

### Runtime execution

**Koras** enables `JSX-like` syntax anywhere JavaScript runs — browser, server, or worker. Now, you can use the same component in clients and servers.

### Synchronous and asynchronous rendering

Components can run synchronously, asynchornously or both at the same time. It enable rendering flexibility.

### No compilation step required

Everything runs entirely at runtime so no compilation is required.

### No virtual DOM

It supports absolute fine-grained rendering and re-rendering via component layering or federation.

### Lightweight, fast rendering model

Koras is less than 23kb (minified) and its rendering and re-rendering model is blazingly fast.

### Agent and subagent (AI and Non-AI) orchestrations

AI or non-AI agent and subagent orchestrations are enabled in the browser with `Lazy Recursion` and flexible components.

### Simple learning curve

Learning **koras** is straightforward as it is vanilla JavaScript in operation.

Koras is designed for direct execution and minimal runtime cost on both client and server.
